# j3DevHelper UI Specification

## 1. 문서 목적

`j3DevHelper` 초기 화면을 구현하기 위한 UI 명세를 정리한다.

## 2. 화면 개요

- 화면 이름: 메인 윈도우
- 애플리케이션 이름: `j3DevHelper`
- 플랫폼: Windows/Linux 데스크톱 애플리케이션
- 기본 용도: 좌측 워크스페이스를 선택하고, 명령 그룹에서 명령을 관리하거나 실행하는 작업 화면
- 초기 상태: 등록된 워크스페이스와 명령 버튼 내용은 비어 있다.

## 3. 윈도우

| 항목 | 명세 |
| --- | --- |
| 제목 | `j3DevHelper` |
| 윈도우 유형 | 일반 데스크톱 최상위 창 |
| 초기 크기 | 약 484 x 613 px |
| 최소 크기 | 현재 글꼴 크기와 DPI를 기준으로 계산한다. 좌측 트리 최소 폭 72 px, 패널 간격, 우측 명령 그룹 영역 최소 폭 116 px, 메인 패널 최소 높이 160 px를 보장한다. |
| 리사이즈 | 지원. 좌측 패널은 기본 폭에서 시작하며, 사용자가 좌우 영역 사이를 드래그해 폭을 조절할 수 있다. 우측 패널은 남은 폭을 채운다. |
| 배경색 | 기본 시스템 창 배경색 또는 매우 밝은 회색 |
| 타이틀바 | 시스템 기본 타이틀바 사용 |
| 앱 아이콘 | Windows는 `icon.ico`, Linux GTK4는 `icon.svg`를 우선 프로그램 창 아이콘으로 표시하고 없으면 `icon.png`를 사용 |
| 실행 파일 아이콘 | Windows 빌드에서는 `icon.ico`를 실행 파일 아이콘으로 적용. Linux 배포에서는 데스크톱 파일/아이콘 설치 정책에 맞춰 `icon.svg` 또는 `icon.png`를 사용 |

아이콘 명세:

- Windows 아이콘 파일명은 `icon.ico`로 고정한다.
- Windows에서는 `icon.ico`를 실행 파일 아이콘과 프로그램 창 아이콘에 같은 자산으로 적용한다.
- Linux GTK4에서는 `icon.svg`를 창 아이콘으로 우선 사용하고 없으면 `icon.png`를 사용한다. `--install`은 `io.github.edgarp9.j3DevHelper` desktop entry와 hicolor 아이콘 설치 경로를 함께 맞추며, Plasma fallback용 `io.github.edgarp9.j3devhelper` alias도 설치한다.
- Windows 작업 표시줄과 Alt+Tab 전환 화면에서도 동일한 아이콘이 표시되어야 한다.
- `icon.svg`와 그 변환/패키징 파생물인 `icon.ico`는 Google Fonts Icons / Material Symbols `logo_dev` 기반 SIL Open Font License 1.1 (OFL-1.1) 자산으로 고지하며, SVG/ICO 파일에 대한 정확한 적용 여부는 `THIRD_PARTY_NOTICES.txt`의 확인 필요 항목에 남긴다.

## 4. 전체 레이아웃

화면은 위에서 아래 순서로 다음 영역을 가진다.

1. 메뉴 바
2. 메인 콘텐츠 영역

메인 콘텐츠 영역은 좌우 2열 구조다.

| 영역 | 위치 | 명세 |
| --- | --- | --- |
| 좌측 영역 | 왼쪽 | 워크스페이스 트리 패널 |
| 분할 조절 영역 | 가운데 | 좌우 영역 사이 간격. 마우스를 올리면 좌우 리사이즈 커서로 바뀌고 드래그로 좌측 폭을 조절한다. |
| 우측 영역 | 오른쪽 | 명령 그룹 드롭다운, 명령 버튼 페이지 |

권장 여백:

- 창 내부 좌우 여백: 약 8 px
- 메뉴 바와 콘텐츠 사이 간격: 약 6 px
- 좌측 패널과 우측 패널 사이 간격: 약 10 px

## 5. 메뉴 바

메뉴 바는 윈도우 상단에 배치한다.

| 순서 | 메뉴 텍스트 | 하위 항목 | 역할 |
| --- | --- | --- | --- |
| 1 | `File`/`파일` | `Font`/`글꼴`, `Theme`/`테마`, `UI Language`/`UI 언어`, `Workspace Languages`/`워크스페이스 언어`, `About`/`정보`, `Exit`/`종료` | 표시 설정, UI 언어 선택, 워크스페이스 언어 목록 설정, 앱 정보 확인, 애플리케이션 종료 |
| 2 | `워크스페이스` | `추가`, `분류 추가`, `편집`, `위로`, `아래로`, `삭제` | 좌측 워크스페이스 트리의 노드와 분류 관리 |
| 3 | `명령 그룹` | `추가`, `이름 변경`, `위로`, `아래로`, `삭제` | 우측 명령 그룹 관리 |
| 4 | `명령` | `실행`, `추가`, `편집`, `앞으로`, `뒤로`, `삭제` | 선택된 명령 그룹 안의 명령 버튼 관리 |

명세:

- 메뉴 텍스트는 좌측에서 우측으로 정렬한다.
- 메뉴 높이는 시스템 기본 메뉴 바 높이를 따른다.
- 메뉴의 정보 구조는 작업 대상 기준으로 구분한다.
  - `File`/`파일`: 애플리케이션 전체에 적용되는 표시 설정, UI 언어, 워크스페이스 언어 목록 설정, 앱 정보, 종료
  - `워크스페이스`: 워크스페이스, 폴더, 노드, 분류처럼 좌측 트리 안에서 관리되는 대상
  - `명령 그룹`: 명령을 묶어 드롭다운에서 선택하는 단위
  - `명령`: 명령 그룹 내부에 배치되는 실행 버튼
- 선택된 대상이 없으면 해당 대상에 의존하는 메뉴 항목은 비활성화한다.
- 편집, 삭제, 위로/아래로처럼 기존 대상을 변경하는 항목은 현재 선택 대상이 명확할 때만 활성화한다.
- `파일 > 종료`는 메인 창에 닫기 요청을 보내고 표준 종료 경로를 따른다.
- `파일 > 정보`는 앱 이름과 버전을 한 줄(`j3DevHelper  v{version}`)로 표시하고, `https://github.com/edgarp9` 링크와 `Licenses`/`라이선스` 영역에 배포 파일 `about.txt` 내용을 표시한다. `about.txt`에는 프로젝트 GPL-3.0-or-later, 보증 없음, `LICENSE`, `THIRD_PARTY_NOTICES.txt`, 대응 소스 코드 위치, 제3자 고지 요약이 포함된다. 설치된 앱에서 `about.txt`를 찾지 못하면 빌드에 포함된 기본 텍스트를 표시한다. 라이선스 영역은 긴 고지 목록을 확인할 수 있도록 세로 스크롤을 제공한다.

인지 UX 기준:

- 사용자는 먼저 작업 대상을 고르고 메뉴에서 동작을 선택하는 흐름을 따른다.
- 파괴적 동작인 `삭제`는 즉시 실행하지 않고 확인 절차를 둔다.
- 삭제와 종료는 실수 클릭을 줄이기 위해 각 메뉴의 뒤쪽에 둔다.
- `워크스페이스`와 `명령 그룹`의 `위로`와 `아래로`는 선택된 대상을 한 칸씩 이동하며, 이동 가능한 방향만 활성화한다.
- `이름 변경`은 명령 그룹에만 사용하고, 트리와 명령 버튼에는 더 넓은 속성 수정을 의미하는 `편집`을 사용한다.
- `명령 그룹 > 추가`와 `명령 > 추가`는 최상위 메뉴명으로 추가 대상을 구분한다.

### 5.1 `파일 > 글꼴`

`파일 > 글꼴`은 애플리케이션 전체 UI 글꼴과 글꼴 크기를 변경하는 설정 화면을 연다.

설정 대상:

- 좌측 트리 항목
- 우측 명령 그룹 드롭다운 텍스트
- 명령 버튼 텍스트
- 대화상자와 설정 화면의 일반 UI 텍스트

타이틀바와 표준 메뉴 바처럼 OS가 직접 그리는 영역은 시스템 정책을 따른다.

글꼴 설정 화면:

| 항목 | 명세 |
| --- | --- |
| 글꼴 선택 | 시스템에 설치된 글꼴 목록에서 선택 |
| 글꼴 크기 | 숫자 입력 또는 선택 목록으로 변경 |
| 미리보기 | 선택한 글꼴과 크기를 샘플 문장에 즉시 반영 |
| 적용 | 현재 창에 변경 사항을 반영하고 설정 파일에 저장 |
| 취소 | 변경 사항을 버리고 이전 설정 유지 |
| 기본값 | 기본 글꼴과 기본 크기로 되돌림 |

동작 명세:

- 글꼴 목록은 실행 중인 OS에서 사용 가능한 설치 글꼴을 기준으로 표시한다.
- 글꼴 목록은 설치 글꼴이 많은 환경에서도 세로 스크롤로 모든 항목을 선택할 수 있어야 한다.
- 글꼴 이름은 사용자가 직접 경로를 입력하지 않고 목록에서 선택한다.
- 글꼴 크기는 UI가 깨지지 않는 범위인 `9` pt 이상 `20` pt 이하에서만 허용한다.
- 글꼴 크기 선택 목록은 `9`, `10`, `11`, `12`, `13`, `14`, `16`, `18`, `20`을 제공한다.
- 변경 적용 후 창 레이아웃을 다시 계산해 텍스트가 버튼, 드롭다운, 트리 항목 밖으로 넘치지 않게 한다.
- 선택한 글꼴이 다음 실행 시 시스템에 없으면 기본 글꼴로 대체하고 사용자에게 설정 복원 실패를 안내한다.
- Linux에서 기본값 `Segoe UI` 자체가 설치되어 있지 않은 경우는 Windows 기본 설정과 같은 정상 시작으로 보고, 별도 복원 경고 없이 GTK/Pango fallback 글꼴을 사용한다.
- 이 경우 Linux 글꼴 설정 대화상자는 `Segoe UI`를 현재 선택값으로 유지해, 사용자가 아무 변경 없이 적용해도 첫 번째 설치 글꼴로 설정이 바뀌지 않게 한다.
- 글꼴 이름과 글꼴 크기는 설정 파일에 저장한다.

### 5.2 `파일 > 테마`

`파일 > 테마`는 하위 메뉴에서 애플리케이션 표시 테마를 즉시 선택한다.

테마 선택지:

| 저장값 | 메뉴 텍스트 | 적용 기준 |
| --- | --- | --- |
| `system` | `시스템` | OS 기본 색과 컨트롤 렌더링 |
| `light` | `밝게` | j3TreeText 밝은 테마 계열 |
| `classic-dark` | `어둡게` | j3TreeText 어두운 기본 계열 |
| `sepia-teal` | `세피아` | j3TreeText 세피아/틸 계열 |
| `graphite` | `그래파이트` | j3TreeText 그래파이트 계열 |
| `forest` | `숲` | j3TreeText 포레스트 계열 |
| `steel-blue` | `스틸 블루` | j3TreeText 스틸 블루 계열 |

동작 명세:

- 선택한 테마는 메뉴에서 체크 상태로 표시한다.
- 기본 테마는 `graphite`이며, 설정 파일에 테마가 없거나 지원하지 않는 값이면 `graphite`로 복원한다.
- 선택 즉시 창 배경, 좌측 TreeView, 우측 명령 버튼 콘텐츠 배경, 우측 버튼 컨트롤, 타이틀바의 다크 모드 선호값을 갱신한다.
- 선택한 테마는 `view.theme`에 저장하며, 다음 실행 시 복원한다.
- 기존 설정값 `dark`는 `classic-dark`로 복원한다.

### 5.3 `파일 > UI 언어`

`파일 > UI 언어`는 하위 메뉴에서 애플리케이션 표시 언어를 즉시 선택한다.

UI 언어 선택지:

| 저장값 | 한국어 UI 메뉴 텍스트 | 영어 UI 메뉴 텍스트 |
| --- | --- | --- |
| `ko` | `한국어` | `Korean` |
| `en` | `영어` | `English` |

동작 명세:

- 기본 UI 언어는 영어 UI 적용 상태인 `en`이다.
- 선택한 UI 언어는 메뉴에서 체크 상태로 표시한다.
- 선택 즉시 메뉴 바와 이후 열리는 대화상자/메시지의 표시 언어를 갱신한다.
- 선택한 UI 언어는 `view.ui_language`에 저장하며, 다음 실행 시 복원한다.
- 설정 파일에 UI 언어가 없거나 지원하지 않는 값이면 기본값 `en`으로 복원한다.

### 5.4 `파일 > 워크스페이스 언어`

`파일 > 워크스페이스 언어`는 워크스페이스 `Language` 선택 목록을 변경하는 설정 화면을 연다.

설정 대상:

- `워크스페이스 > 추가`와 `워크스페이스 > 편집`에서 표시되는 `Language` 콤보박스
- 폴더 드래그앤드롭 등록 시 자동 추정 결과를 적용할 수 있는 언어 목록
- 설정 파일의 `languages` 배열

편집 화면:

| 항목 | 명세 |
| --- | --- |
| 언어 목록 | 줄바꿈 또는 쉼표로 구분된 언어 목록 |
| 저장 | 유효성 검사를 통과하면 저장하고 창을 닫음 |
| 취소 | 변경 사항을 버리고 창을 닫음 |
| 기본값 | 기본 언어 목록으로 되돌림 |

동작 명세:

- 기본 언어 목록은 `Rust`, `TypeScript`, `JavaScript`, `Python`, `Go`, `Java`, `C#`, `C`, `C++`, `Other`다.
- 사용자는 예를 들어 `rust, c, java`처럼 소문자 언어명도 저장할 수 있다.
- 비어 있는 항목은 무시한다.
- 중복 언어명은 대소문자만 다른 경우도 허용하지 않는다.
- 편집 후 현재 워크스페이스가 사용하는 `Language`가 새 목록에 없으면 저장하지 않고 안내한다.
- 대소문자만 변경된 언어명은 기존 워크스페이스 `Language`도 새 목록의 표기로 정규화한다.

## 6. 좌측 영역: Workspace Tree

### 6.1 트리 패널

| 항목 | 명세 |
| --- | --- |
| 컴포넌트 유형 | 트리 뷰 또는 트리 컨테이너 |
| 초기 내용 | 등록된 워크스페이스 없음 |
| 테두리 | 1 px 단색 회색 테두리 |
| 배경색 | 흰색 |
| 권장 폭 | 약 160 px |
| 권장 높이 | 메인 콘텐츠 영역 높이를 채움 |
| 드롭 대상 | 폴더 드래그앤드롭 허용 |
| 주 표시값 | 워크스페이스의 `name` |
| 보조 정보 | `path`, `Language`는 상세 보기, 툴팁, 편집 화면에서 확인 가능 |

초기 상태:

- 선택된 워크스페이스 없음
- 선택된 트리 노드 없음
- 스크롤바는 항목이 없으므로 표시하지 않음
- 저장된 워크스페이스나 분류가 복원된 경우에도 첫 실행 화면에서 임의의 첫 항목을 자동 선택하지 않음

추정 동작:

- 워크스페이스가 등록되면 루트 또는 폴더 노드를 트리 형태로 표시한다.
- 루트 깊이에서는 분류와 최상위 워크스페이스가 같은 순서 목록을 공유하며, 분류 우선 또는 워크스페이스 우선으로 고정하지 않는다.
- 트리 노드 선택 시 우측 명령 그룹 영역의 컨텍스트가 갱신될 수 있다.
- 워크스페이스 또는 분류 노드를 우클릭하면 우클릭한 노드를 선택하고 컨텍스트 메뉴를 표시한다.
- 컨텍스트 메뉴는 `편집`, `위로`, `아래로`, `워크스페이스 추가`, `분류 추가`, `삭제`를 제공한다.
- `삭제`는 워크스페이스 또는 분류 노드에서 활성화한다.
- 분류를 삭제하면 분류만 제거하고, 해당 분류에 속한 워크스페이스는 삭제하지 않고 Tree의 1차 깊이로 이동한다.

### 6.2 워크스페이스 정보 구조

좌측 워크스페이스는 다음 3개 필드로 구성한다.

| 필드 | 필수 여부 | 입력 방식 | UI 표시 원칙 |
| --- | --- | --- | --- |
| `path` | 필수 | 폴더 선택 또는 폴더 드래그앤드롭 | 트리에는 항상 노출하지 않고 상세/툴팁/편집 화면에서 표시 |
| `name` | 필수 | 사용자 입력. 드래그앤드롭 시 폴더명을 기본값으로 사용 | 트리 노드의 주 텍스트로 표시 |
| `Language` | 필수 | 선택 목록 또는 자동 추정 후 사용자 확인 | 트리에서는 필요 시 짧은 보조 표식으로 표시 |

정보 아키텍처 기준:

- `path`는 워크스페이스의 실제 위치와 중복 여부 판단에 사용한다.
- `name`은 사용자가 트리에서 스캔하고 선택하는 주 식별자다.
- `Language`는 명령 그룹, 명령 버튼, 향후 언어별 기본 명령 추천에 사용할 수 있는 분류 정보다.
- 트리 노드의 기본 표시 텍스트는 `name`만 사용해 좌측 패널의 인지 부하를 낮춘다.
- `path`는 길어질 수 있으므로 트리 안에 항상 표시하지 않는다.
- `Language`는 짧은 텍스트나 아이콘 배지로 보조 표시할 수 있지만, 이름보다 시각적으로 강하면 안 된다.

`워크스페이스 > 추가` 입력 화면:

| 항목 | 명세 |
| --- | --- |
| `path` | 폴더 선택 버튼으로 지정 |
| `name` | 기본값은 선택한 폴더명, 사용자가 수정 가능 |
| `Language` | 언어 선택 목록에서 선택. 자동 추정 결과가 있으면 기본 선택값으로 제안 |
| 저장 | 유효성 검사를 통과하면 워크스페이스 등록 |
| 취소 | 입력 내용을 버리고 닫음 |

`워크스페이스 > 편집` 입력 화면:

- 선택된 워크스페이스의 `path`, `name`, `Language`를 표시한다.
- `name`과 `Language`는 사용자가 수정할 수 있다.
- `path` 변경은 기존 워크스페이스의 실제 위치를 바꾸는 작업이므로 폴더 선택 버튼으로만 허용한다.
- `path` 변경 시 기존 등록 항목과 중복되는지 확인한다.
- 분류 노드는 워크스페이스가 아니므로 `path`와 `Language`를 갖지 않는다. 분류 편집은 이름 중심으로 처리한다.

`워크스페이스 > 분류 추가` 입력 화면:

- 분류 `name`만 입력받는다.
- 저장하면 Tree의 1차 깊이에 폴더형 노드를 추가한다.
- 입력칸에서 `Enter`를 누르면 저장 버튼과 같은 동작을 수행한다.
- 분류 안에 다른 분류를 추가하는 중첩 분류는 지원하지 않는다.

유효성 규칙:

- `path`는 비어 있을 수 없고 접근 가능한 폴더여야 한다.
- `name`은 비어 있을 수 없고 같은 부모 아래에서 사용자가 구분 가능한 이름이어야 한다.
- `Language`는 선택 목록의 값이어야 한다. 자동 추정이 실패하면 사용자가 선택할 수 있도록 한다.
- 같은 `path`를 가진 워크스페이스는 중복 등록하지 않는다. 중복 판정은 Windows 기준처럼 경로 양끝 공백과 끝 경로 구분자를 제외하고 ASCII 대소문자를 구분하지 않는다.

인지 UX 기준:

- 등록과 편집 화면에서는 `name`을 먼저 보여주고, `path`는 그 아래에 배치한다.
- `Language`는 명령 구성에 영향을 줄 수 있으므로 저장 전 확인 가능한 위치에 둔다.
- 긴 `path`는 가운데를 줄임 처리하되, 전체 경로는 툴팁이나 읽기 전용 텍스트 선택으로 확인할 수 있어야 한다.

### 6.3 폴더 드래그앤드롭 등록

사용자는 OS 파일 탐색기에서 폴더를 드래그해 좌측 트리 패널에 놓을 수 있다.

동작 명세:

- 드롭 가능한 대상은 폴더다.
- 파일을 드롭한 경우 등록하지 않고 사용자에게 이해 가능한 메시지를 표시한다.
- 여러 폴더를 동시에 드롭할 수 있는지는 미정이다. 초기 구현에서는 단일 폴더 등록을 우선한다.
- 폴더가 정상적으로 드롭되면 `path`, `name`, `Language`를 가진 워크스페이스 노드로 등록한다.
- 드롭된 폴더 경로는 `path`에 저장한다.
- 드롭된 폴더명은 `name`의 기본값으로 사용한다.
- `Language`는 가능한 경우 폴더 내용을 기준으로 자동 추정하고, 추정할 수 없으면 사용자 선택을 요청한다.
- 초기 구현의 언어 자동 추정은 드롭된 폴더의 최상위 항목만 확인한다.
  - `Cargo.toml` 또는 `*.rs`: `Rust`
  - `tsconfig.json`, `*.ts`, `*.tsx`: `TypeScript`
  - `package.json`, `*.js`, `*.jsx`: `JavaScript`
  - `pyproject.toml`, `requirements.txt`, `setup.py`, `*.py`: `Python`
  - `go.mod`, `*.go`: `Go`
  - `pom.xml`, `build.gradle`, `build.gradle.kts`, `*.java`: `Java`
  - `*.csproj`, `*.cs`: `C#`
  - `*.c`: `C`
  - `CMakeLists.txt`, `*.cpp`, `*.cc`, `*.cxx`, `*.hpp`, `*.hxx`, `*.hh`: `C++`
- 자동 추정이 실패하면 `path`와 `name`을 채운 워크스페이스 추가 대화상자를 열고, `Language`는 사용자가 선택할 수 있게 한다.
- 트리의 빈 영역에 드롭하면 최상위 노드로 등록한다.
- 이번 초기 구현에서는 기존 노드 위에 드롭해도 최상위 워크스페이스로 등록한다.
- 이미 등록된 폴더를 다시 드롭하면 중복 등록을 막고 기존 노드를 안내한다.
- 접근 권한이 없거나 경로를 읽을 수 없는 폴더는 등록하지 않는다.

드래그 피드백:

- 드래그 중 트리 패널 위에 진입하면 드롭 가능 상태를 시각적으로 표시한다.
- 등록 가능한 단일 폴더가 트리 패널 위에 있을 때 TreeView 배경을 테마 대비를 유지하는 파랑 계열 피드백 색으로 표시한다.
- 드롭 불가능한 항목이면 금지 커서 또는 테마 대비를 유지하는 빨강 계열 피드백 색으로 표시한다.
- 기존 노드 위 삽입 위치 또는 부모 노드 후보 표시는 하위 노드 등록을 구현할 때 함께 제공한다.

### 6.4 워크스페이스와 분류 이동

사용자는 좌측 Tree에서 워크스페이스 노드를 마우스, 키보드, 메뉴로 이동할 수 있고, 분류 노드는 키보드와 메뉴로 이동할 수 있다.

동작 명세:

- 트리 이동은 워크스페이스 노드와 분류 노드를 대상으로 하되, 분류 노드는 키보드와 `워크스페이스 > 위로/아래로` 메뉴 이동을 지원한다.
- 마우스 드래그 이동 대상은 워크스페이스 노드다. 워크스페이스를 분류 노드 위에 놓으면 해당 분류 하위로 배치한다.
- 최상위 워크스페이스를 다른 최상위 워크스페이스의 위쪽 절반에 놓으면 `tree_order`에서 해당 워크스페이스 앞에 배치한다.
- 최상위 워크스페이스를 다른 최상위 워크스페이스의 아래쪽 절반에 놓으면 `tree_order`에서 해당 워크스페이스 뒤에 배치한다.
- 분류 하위 워크스페이스를 같은 분류 안의 다른 워크스페이스 위 또는 아래에 놓으면 해당 분류 안에서 순서를 바꾼다.
- 드래그 중 유효한 드롭 대상 워크스페이스 또는 분류 노드 위에 있으면 해당 대상 노드 배경을 테마 대비를 유지하는 피드백 색으로 표시한다.
- 대상 노드에서 벗어나거나 드롭 또는 취소되면 피드백 색을 제거하고 기존 트리 항목 색으로 복원한다.
- 최상위 워크스페이스 노드에 포커스가 있을 때 `Ctrl+Up` 또는 `Ctrl+Down`을 누르면 분류와 최상위 워크스페이스가 섞인 루트 순서 안에서 한 칸 위/아래로 이동한다.
- 분류 하위 워크스페이스 노드에 포커스가 있을 때 `Ctrl+Up` 또는 `Ctrl+Down`을 누르면 같은 분류 안에서 한 칸 위/아래로 이동한다.
- 분류 노드에 포커스가 있을 때 `Ctrl+Up` 또는 `Ctrl+Down`을 누르면 분류와 최상위 워크스페이스가 섞인 루트 순서 안에서 한 칸 위/아래로 이동한다.
- `워크스페이스 > 위로`와 `워크스페이스 > 아래로`는 현재 선택된 워크스페이스 또는 분류에 동일한 한 칸 이동 규칙을 적용한다.
- 자기 자신 또는 인접한 현재 위치에 놓는 이동은 순서 변경 없이 취소한다.
- 이동이 완료되면 최상위 노드는 변경된 `tree_order`를, 분류 하위 워크스페이스는 변경된 `workspaces` 순서를 설정 파일에 저장하고, 이동한 노드를 선택 상태로 유지한다.


## 7. 우측 영역: 명령 그룹

### 7.1 명령 그룹 드롭다운

| 항목 | 명세 |
| --- | --- |
| 컴포넌트 유형 | 드롭다운 목록 |
| 초기 내용 | 비어 있음 |
| 테두리 | 시스템 기본 콤보박스 테두리 |
| 권장 폭 | 남은 수평 영역 전체 |
| 권장 높이 | 한 줄 선택 컨트롤 높이. 펼침 목록은 사용 가능한 우측 영역 안에서 표시 |

초기 상태:

- 명령 그룹이 없으면 드롭다운은 빈 상태로 비활성화한다.

추정 동작:

- 명령 그룹은 명령 분류를 나타낸다.
- 사용자는 드롭다운에서 현재 명령 그룹을 선택한다.
- `명령 그룹` 메뉴 또는 `명령` 메뉴를 통해 명령 그룹과 명령을 관리할 수 있다.
- `명령 그룹 > 위로`는 선택된 명령 그룹을 드롭다운 순서에서 한 칸 위로 옮기고, 첫 번째 명령 그룹에서는 비활성화한다.
- `명령 그룹 > 아래로`는 선택된 명령 그룹을 드롭다운 순서에서 한 칸 아래로 옮기고, 마지막 명령 그룹에서는 비활성화한다.

### 7.2 명령 버튼

명령 버튼은 선택된 `명령 그룹` 안에 배치되는 실행 단위다.

| 항목 | 명세 |
| --- | --- |
| 표시 텍스트 | 명령 버튼의 `button_name` |
| 소속 | 현재 선택된 명령 그룹 |
| 실행 조건 | 실행 가능한 워크스페이스 컨텍스트가 필요하면 좌측 워크스페이스가 선택되어 있어야 함 |
| 실행 진입 | 명령 버튼 클릭 또는 `명령 > 실행` |
| 편집 진입 | `명령 > 추가` 또는 `명령 > 편집` |
| 삭제 진입 | 명령 버튼 선택 후 `명령 > 삭제` |
| 이동 진입 | 명령 버튼 선택 후 `명령 > 앞으로` 또는 `명령 > 뒤로` |

초기 상태:

- 선택된 명령 그룹에 명령 버튼이 없으면 빈 콘텐츠 영역을 표시한다.
- 명령 버튼이 선택되지 않은 상태에서는 `명령 > 실행`, `명령 > 편집`, `명령 > 앞으로`, `명령 > 뒤로`, `명령 > 삭제`를 비활성화한다.
- `명령 > 앞으로`는 선택된 명령 버튼을 같은 명령 그룹 안에서 한 칸 앞 위치로 이동하며, 첫 번째 명령 버튼에서는 비활성화한다.
- `명령 > 뒤로`는 선택된 명령 버튼을 같은 명령 그룹 안에서 한 칸 뒤 위치로 이동하며, 마지막 명령 버튼에서는 비활성화한다.
- 이번 구현에서 명령 버튼 클릭은 현재 선택된 워크스페이스와 명령 버튼을 확인한 뒤 버튼 실행으로 처리한다.
- 클릭된 명령 버튼은 실행 전 선택 상태로 반영해 `명령 > 실행`, `명령 > 편집`, `명령 > 앞으로`, `명령 > 뒤로`, `명령 > 삭제`의 대상도 명확히 한다.
- 선택된 명령 그룹이 없으면 `명령 > 추가`도 비활성화한다.
- 선택된 명령 그룹 안의 명령 버튼은 드롭다운 아래 콘텐츠 영역에 플랫폼 네이티브 컨트롤 목록으로 표시한다. Windows는 Win32 `BUTTON` 컨트롤 격자, Linux는 GTK4 `FlowBox` wrapping grid를 사용한다. 버튼 폭, 높이, 내부 시작 여백, 행/열 간격은 현재 UI 글꼴 크기에 맞춰 같은 기본 규칙으로 조정한다.
- 명령 버튼은 명령 그룹 드롭다운 아래 콘텐츠 영역에 배치한다.
- 명령 버튼 목록의 전체 높이가 콘텐츠 영역보다 크면 세로 스크롤로 숨겨진 버튼까지 탐색한다.
- 세로 스크롤의 마지막 위치에서는 마지막 명령 버튼 행 전체가 보여야 하며, 필요한 경우 페이지 하단에 빈 여백을 남긴다.
- 명령 버튼 폭은 96 DPI 기준 선호 폭 132 px를 사용하되, 우측 영역이 좁으면 최소 96 px까지 줄어들 수 있다.
- 긴 `button_name`은 버튼 안에서 왼쪽부터 표시하고, 버튼 폭을 넘는 오른쪽 텍스트는 숨긴다.
- 긴 명령 버튼 텍스트는 같은 행의 오른쪽 버튼 텍스트나 영역을 덮지 않아야 한다.
- 명령 버튼 hover 툴팁은 해당 버튼의 전체 `button_name`을 표시한다.
- 명령 버튼을 우클릭하면 해당 버튼을 선택하고 컨텍스트 메뉴를 표시한다.
- 컨텍스트 메뉴는 `실행`, `편집`, `앞으로`, `뒤로`, `명령 추가`, `삭제`를 제공한다.
- 좌측 Tree에서 분류 노드가 선택된 상태에서는 컨텍스트 메뉴의 `실행`을 비활성화한다.

### 7.2.1 명령 버튼 마우스 이동

사용자는 선택된 명령 그룹 안의 명령 버튼을 마우스로 끌어 같은 명령 그룹 안의 다른 명령 버튼 위에 놓을 수 있다.

동작 명세:

- 이동 대상은 현재 선택된 명령 그룹 안의 명령 버튼이다.
- 버튼을 다른 버튼 위에 놓으면 드롭 대상 버튼이 가리키는 순서 위치로 이동한다.
- 드래그 중 유효한 드롭 대상 버튼 위에 있으면 해당 대상 버튼 배경을 테마 대비를 유지하는 피드백 색으로 표시한다.
- 대상 버튼에서 벗어나거나 드롭 또는 취소되면 피드백 색을 제거하고 기존 버튼 색으로 복원한다.
- 자기 자신 위에 놓거나 다른 버튼 위에 놓지 않은 드롭은 순서 변경 없이 취소한다.
- 드래그로 판단되지 않는 일반 클릭은 기존처럼 명령 버튼 실행으로 처리한다.
- 이동이 완료되면 변경된 `buttons` 순서를 설정 파일에 저장하고, 이동한 명령 버튼을 선택 상태로 유지한다.

### 7.3 버튼 편집창

`명령 > 추가`와 `명령 > 편집`은 같은 버튼 편집창을 사용한다.

| 항목 | 입력 방식 | 필수 여부 | 설명 |
| --- | --- | --- | --- |
| 이름 | 텍스트 입력 | 필수 | 명령 버튼에 표시되는 이름 |
| 실행 대상 | 텍스트 입력 + 파일 선택 버튼 | 필수 | 실행할 프로그램 경로 또는 PATH에서 찾을 명령 |
| 인수 | 텍스트 입력 | 선택 | 실행 시 전달할 인수 문자열 |
| 실행 방식 | 라디오 버튼 또는 선택 목록 | 필수 | 직접 실행 또는 터미널 실행 중 선택 |

편집창 하단 버튼:

| 버튼 | 동작 |
| --- | --- |
| 저장 | 유효성 검사를 통과하면 저장하고 창을 닫음 |
| 취소 | 변경 사항을 버리고 창을 닫음 |
| 적용 | 창을 닫지 않고 현재 값을 저장 |

유효성 규칙:

- 이름은 비어 있을 수 없다.
- 실행 대상은 비어 있을 수 없다.
- `shell_api` 실행 방식에서는 실행 대상이 OS Shell API로 실행 가능한 값이어야 한다.
- `external_terminal` 실행 방식에서는 버튼 실행 시 선택된 워크스페이스의 `path`가 접근 가능한 폴더여야 한다.
- 인수에 알 수 없는 `{...}` 토큰이 있으면 저장 전에 사용자에게 확인 또는 오류를 표시한다.
- 이번 구현에서는 알 수 없는 `{...}` 토큰을 오류로 표시하고 저장하지 않는다.
- 이번 구현에서 `적용`은 선택된 명령 그룹의 명령 버튼 목록과 설정 파일에 즉시 반영한다.
- `취소`는 아직 적용하지 않은 편집 내용만 버리며, 이미 `적용`한 값은 유지한다.

정보 아키텍처 기준:

- 이름은 사용자가 명령 그룹 안에서 명령을 스캔하는 주 식별자이므로 가장 위에 배치한다.
- 실행 대상과 인수는 실행 결과를 결정하는 세부 정보이므로 같은 그룹에 배치한다.
- 실행 방식은 실행 결과가 보이는 방식과 작업 디렉터리를 바꾸므로 인수 아래에서 명확히 선택하게 한다.
- 위험한 실행을 막기 위해 저장 시점보다 실행 시점에 워크스페이스와 치환값을 다시 검증한다.

인지 UX 기준:

- 사용자가 치환 문자열을 외우지 않아도 되도록 인수 입력 영역 옆 또는 아래에 토큰 삽입 버튼을 제공한다.
- 토큰 버튼을 누르면 현재 커서 위치에 해당 문자열을 삽입한다.
- 토큰 삽입 후 인수 입력 영역에 포커스를 유지한다.
- 긴 경로는 가운데를 줄임 처리하되 전체 경로는 툴팁이나 선택 가능한 텍스트로 확인할 수 있어야 한다.

### 7.4 실행 방식

| 실행 방식 값 | UI 표시 | 실행 방식 | 워크스페이스 의존성 |
| --- | --- | --- | --- |
| `shell_api` | 직접 | OS Shell API로 실행 대상을 바로 실행하고 치환된 인수를 전달 | `{path}`, `{name}`, `{Language}` 토큰을 쓰는 경우 선택된 워크스페이스 필요 |
| `external_terminal` | 터미널 | 외부 터미널을 열고 선택된 워크스페이스의 `path`를 작업 디렉터리로 사용해 실행 대상과 치환된 인수를 실행 | 항상 선택된 워크스페이스 필요 |

실행 규칙:

- 버튼 클릭 시 먼저 현재 선택된 워크스페이스를 확인한다.
- 인수의 치환 토큰을 실제 값으로 대체한다.
- 경로에 공백이 있어도 하나의 값으로 전달되도록 실행 계층에서 안전하게 quoting 또는 escaping을 처리한다.
- `shell_api` 실행은 별도 터미널 창을 띄우지 않는다.
- `external_terminal` 실행은 외부 터미널 창을 띄워 사용자가 명령 출력과 종료 상태를 볼 수 있게 한다.
- 실행 실패는 조용히 무시하지 않고 사용자에게 이해 가능한 메시지로 안내한다.
- 외부 터미널 종류와 명령 완료 후 터미널 창 유지 여부는 미정이다.

### 7.5 인수 치환 문자열

인수에는 다음 치환 문자열을 사용할 수 있다.

| 토큰 | 대치 값 | 실행 시 동작 |
| --- | --- | --- |
| `{path}` | 선택된 워크스페이스의 `path` | 워크스페이스가 없으면 실행 취소 |
| `{name}` | 선택된 워크스페이스의 `name`. 기본값은 `path`의 마지막 폴더명이며 사용자가 편집 가능 | 워크스페이스가 없으면 실행 취소 |
| `{selectfile}` | 파일 선택창에서 선택한 파일의 경로 | 파일 선택창을 열고, 취소하면 실행 취소 |
| `{selectdir}` | 폴더 선택창에서 선택한 폴더의 경로 | 폴더 선택창을 열고, 취소하면 실행 취소 |
| `{inputtext}` | 텍스트 입력창에서 입력한 문자열 | 텍스트 입력창을 열고, 취소하면 실행 취소 |
| `{Language}` | 선택된 워크스페이스의 `Language` | 워크스페이스가 없으면 실행 취소 |

토큰 삽입 버튼:

- 명령 추가/편집 대화상자에서는 토큰 삽입 버튼을 3열 x 2행으로 배치한다.
- 각 토큰 버튼은 `{Language}`처럼 긴 토큰도 잘리지 않는 폭을 확보한다.

| 버튼 텍스트 | 삽입 문자열 |
| --- | --- |
| `{path}` | `{path}` |
| `{name}` | `{name}` |
| `{selectfile}` | `{selectfile}` |
| `{selectdir}` | `{selectdir}` |
| `{inputtext}` | `{inputtext}` |
| `{Language}` | `{Language}` |

치환 규칙:

- 같은 토큰이 여러 번 있으면 한 번 얻은 값을 모든 동일 토큰 위치에 동일하게 대치한다.
- `{selectfile}`, `{selectdir}`, `{inputtext}`가 함께 있으면 인수 문자열에 처음 등장하는 순서대로 사용자 입력을 요청한다.
- 사용자가 중간에 선택 또는 입력을 취소하면 실행 전체를 취소한다.
- `{Language}`는 대소문자를 구분한다. `{language}`는 유효한 토큰이 아니다.
- 치환 후 비어 있는 인수는 빈 문자열로 전달한다.
- 토큰은 인수 필드에서만 대치한다. 이름과 실행 대상에서는 토큰 치환을 하지 않는다.

## 8. 시각 스타일

| 항목 | 명세 |
| --- | --- |
| 전체 톤 | 플랫폼 기본 데스크톱 애플리케이션 스타일 |
| 글꼴 | 기본값은 시스템 기본 UI 글꼴, 사용자가 선택한 설치 글꼴로 변경 가능 |
| 글꼴 크기 | 기본값은 시스템 기본 UI 크기, 사용자가 변경 가능 |
| 라벨 크기 | 선택된 UI 글꼴 크기를 기준으로 적용 |
| 메뉴 텍스트 크기 | 시스템 기본 메뉴 글꼴 또는 사용자가 선택한 UI 글꼴 정책을 따른다 |
| 패널 테두리 | 회색 1 px |
| 좌측 패널 배경 | 흰색 |
| 우측 명령 버튼 콘텐츠 배경 | 밝은 회색 |

구현 시 과도한 장식, 카드형 레이아웃, 그라데이션 배경은 사용하지 않는다.

## 9. 반응형 및 리사이즈 규칙

리사이즈를 지원하는 경우 다음 규칙을 권장한다.

- 좌측 `워크스페이스` 영역의 폭은 기본 160 px 내외로 시작한다.
- 사용자는 좌측 `워크스페이스`와 우측 `명령 그룹` 사이의 간격을 마우스로 드래그해 좌측 폭을 조절할 수 있다.
- 드래그 중 좌측 트리 영역은 최소 폭 이하로 줄어들지 않고, 우측 명령 그룹 영역도 최소 폭 이하로 줄어들지 않는다.
- 우측 `명령 그룹` 영역은 남은 폭을 채운다.
- 창 높이가 늘어나면 좌측 트리 패널과 우측 명령 버튼 콘텐츠 높이가 함께 늘어난다.
- 창 폭이 너무 좁아지면 우측 영역이 먼저 줄어들고, 필요하면 좌측 `워크스페이스` 영역도 최소 폭까지 줄어든다. 최소 폭 이하에서는 더 이상 축소하지 않는다.
- 글꼴 크기가 변경되면 각 패널의 최소 크기와 메뉴 높이를 다시 계산한다.
- 큰 글꼴 크기에서도 메뉴 텍스트, 드롭다운 텍스트, 명령 버튼 텍스트가 겹치지 않아야 한다.
- Windows 커스텀 팝업/대화상자는 현재 UI 글꼴 크기, 실제 적용된 Win32 `HFONT` 높이, DPI를 함께 반영해 외곽 크기와 내부 컨트롤 위치/크기를 계산한다. `20` pt에서도 라벨, 입력칸, 버튼이 서로 겹치거나 잘리지 않아야 하며, debug/release 또는 설정 파일 위치 차이로 실제 폰트가 설정값보다 커진 경우에도 실제 폰트를 기준으로 확장한다.
- Windows 실행 파일은 Per-Monitor DPI Awareness V2 매니페스트를 포함한다. 초기 메인 창 폭/높이는 저장된 실제 창 픽셀 또는 기본 px 값을 그대로 사용하고, 현재 창 DPI는 메인 레이아웃, 대화상자 좌표, 글꼴 픽셀 높이 계산에 반영한다.
- Windows에서 `WM_DPICHANGED`가 오면 새 DPI와 Windows 권장 창 크기를 적용한 뒤 `WM_SETFONT`와 `WM_SIZE` 기반 레이아웃 재계산을 다시 수행한다.
- Windows에서 `WM_GETMINMAXINFO`로 현재 글꼴 크기와 DPI에 맞춘 최소 창 크기를 제공해 좌측 트리와 우측 명령 그룹 영역이 동시에 유지되게 한다.
- Linux GTK4에서는 GTK의 스케일링과 위젯 크기 협상을 따르되, 좌측 트리와 우측 명령 그룹 영역의 최소 폭과 스크롤 가능성을 유지한다.

현재 구현에서는 메뉴 바 아래 클라이언트 영역에 좌측 워크스페이스 트리, 우측 명령 그룹 드롭다운, 명령 버튼 표시 영역을 배치한다. Windows는 `SysTreeView32`와 앱 소유 페이지 창을, Linux는 워크스페이스 트리용 GTK4 `ListBox`, 명령 버튼용 `FlowBox`, `ComboBoxText`, `Paned`를 사용한다. 좌우 영역 상단에 별도 라벨은 표시하지 않는다. 저장된 좌측 폭이 없으면 좌측 컨트롤 폭은 96 DPI 기준 160 px에서 시작해 글꼴 크기와 현재 DPI 또는 GTK 스케일링에 맞춰 조정하고, 저장된 좌측 폭이 있으면 OS가 보고한 실제 px 값 그대로 복원한다.

## 10. 설정 파일

사용자 설정은 기본적으로 프로그램 실행 파일과 같은 폴더에 TOML 파일로 저장한다. 실행 파일 뒤에 설정 파일 경로 인자 하나를 넘기면 해당 TOML 파일을 사용자 설정 파일로 사용한다.

| 항목 | 명세 |
| --- | --- |
| 저장 위치 | 기본값은 실행 파일이 있는 폴더. CLI 설정 파일 경로 인자가 있으면 해당 경로 |
| 파일 이름 | 기본값은 `실행 파일이름.toml` |
| 예시 | Windows 기본값은 `j3DevHelper.exe`와 같은 폴더의 `j3DevHelper.toml`, Linux 기본값은 `j3devhelper`와 같은 폴더의 `j3devhelper.toml`. CLI 예시는 `j3devhelper ./j3devhelper-linux.toml` |
| 형식 | TOML |
| 생성 시점 | 사용자가 저장 가능한 설정을 처음 변경할 때 |
| 읽기 시점 | 프로그램 시작 시 |

저장 대상:

- 선택한 UI 글꼴 이름
- 선택한 UI 글꼴 크기
- 선택한 테마
- 선택한 UI 언어
- 사용자 편집 언어 목록
- 등록된 워크스페이스 목록
- 명령 그룹 목록
- 명령 버튼 정의
- 이후 추가될 화면 표시 설정

권장 TOML 구조:

```toml
[view]
font_family = "Segoe UI"
font_size = 12
theme = "graphite"
ui_language = "en"

languages = ["Rust", "TypeScript", "JavaScript", "Python", "Go", "Java", "C#", "C", "C++", "Other"]

[[categories]]
name = "Backend"

[[tree_order]]
type = "category"
name = "Backend"

[[tree_order]]
type = "workspace"
path = 'C:\dev\scratch'

[[workspaces]]
path = 'C:\dev\j3DevHelper'
name = "j3DevHelper"
Language = "Rust"
category = "Backend"

[[workspaces]]
path = 'C:\dev\scratch'
name = "scratch"
Language = "Other"

[[command_tabs]]
name = "common"

[[command_tabs.buttons]]
button_name = "Open Workspace"
executable_path = 'C:\Windows\explorer.exe'
arguments = "{path}"
execution_type = "shell_api"

[[command_tabs.buttons]]
button_name = "Cargo Check"
executable_path = "cargo"
arguments = "check"
execution_type = "external_terminal"
```

동작 명세:

- 실행 파일 경로가 `C:\Tools\j3DevHelper.exe`이고 설정 파일 경로 인자가 없으면 설정 파일 경로는 `C:\Tools\j3DevHelper.toml`이다.
- 설정 파일 경로 인자가 없을 때 설정 파일 이름은 실행 파일의 확장자를 제거한 이름을 기준으로 만든다.
- 설정 파일 경로 인자가 하나 있으면 해당 경로를 읽고, 이후 설정 변경도 같은 경로에 저장한다.
- 설정 파일 경로 인자가 두 개 이상 있으면 시작 오류로 처리한다.
- 설정 파일이 없으면 기본 설정으로 시작하고, 설정 변경 시 새로 생성한다.
- 설정 파일을 읽을 수 없거나 TOML 파싱에 실패하면 기본 설정으로 시작하고 사용자에게 설정을 적용하지 못했음을 안내한다.
- 설정 파일에 존재하지 않는 항목은 기본값을 사용한다.
- 설정 저장 실패는 조용히 무시하지 않고 사용자에게 안내한다.
- `view.ui_language` 항목이 없거나 `en`/`ko`로 복원할 수 없으면 기본값 `en`을 사용하고 사용자에게 안내한다.
- `languages` 항목이 없으면 기본 언어 목록을 사용한다.
- `languages` 항목이 문자열 배열이 아니거나 비어 있거나 중복 값이 있으면 기본 언어 목록을 사용하고 사용자에게 안내한다.
- `workspaces` 항목을 읽을 때 `path`, `name`, `Language` 중 필수 필드가 없으면 해당 워크스페이스는 건너뛰고 사용자에게 복원되지 않은 항목이 있음을 안내한다.
- `Language` 키는 사용자 요구 필드명과 동일하게 대문자를 유지한다.
- `command_tabs.buttons` 항목을 읽을 때 `button_name`, `executable_path`, `execution_type` 중 필수 필드가 없으면 해당 버튼은 건너뛰고 사용자에게 복원되지 않은 항목이 있음을 안내한다.
- `execution_type`은 `shell_api` 또는 `external_terminal`만 허용한다.
- `external_terminal`의 `executable_path`는 전체 실행 파일 경로를 우선 사용한다. PATH에서 찾을 수 있는 명령 이름만 허용할지는 구현 정책으로 결정한다.

## 11. 접근성

- 메뉴 항목은 키보드 탐색이 가능해야 한다.
- 트리 항목과 명령 그룹 드롭다운 항목은 포커스 표시가 가능해야 한다.
- 색상만으로 상태를 구분하지 않는다.
- 글꼴 크기 변경 후에도 키보드 포커스 표시와 선택 표시가 명확해야 한다.
