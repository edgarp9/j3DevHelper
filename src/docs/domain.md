# j3DevHelper Domain Notes

## 목적

`j3DevHelper`는 워크스페이스 단위의 폴더 또는 작업 컨텍스트를 선택하고,
선택된 컨텍스트에서 사용할 명령 그룹을 관리하는 데스크톱 도구로 정의한다.

현재 문서는 스크린샷 기반 UI 명세를 코드 구현 전에 정리하기 위한 최소 도메인 노트다.
상세 화면 구조는 [UI Specification](ui-spec.md)를 따른다.

## 핵심 용어

| 용어 | 의미 |
| --- | --- |
| Workspace | 사용자가 관리하거나 작업 대상으로 삼는 폴더 기반 작업 단위 |
| Workspace Path | 워크스페이스가 가리키는 실제 폴더 경로. 필드명은 `path`를 사용한다. |
| Workspace Name | 트리에서 사용자에게 표시되는 워크스페이스 이름. 필드명은 `name`을 사용한다. |
| Workspace Language | 워크스페이스의 주요 개발 언어 또는 작업 언어. 필드명은 `Language`를 사용한다. |
| 언어 | 사용자가 워크스페이스 `Language` 선택 목록을 편집하는 설정 |
| Tree | 워크스페이스와 그 하위 항목을 트리 형태로 보여주는 영역 |
| Category | Tree의 1차 깊이에만 추가되는 폴더형 분류 노드. `name`만 가지며 중첩 분류는 지원하지 않는다. |
| Tree Root Order | Tree의 1차 깊이에 표시되는 분류와 최상위 워크스페이스의 혼합 순서. 필드명은 `tree_order`를 사용한다. |
| 명령 그룹 | 명령을 그룹화해 드롭다운에서 선택하는 단위. 저장 키와 내부 타입명은 기존 호환을 위해 `command_tabs`/`CommandTab`을 유지한다. |
| 명령 | 사용자가 클릭해 외부 프로그램이나 명령을 실행하는 버튼 |
| Button Name | 명령 버튼에 표시되는 이름. 내부 필드명은 `button_name`을 사용한다. |
| Executable Path | 실행할 프로그램 또는 명령의 경로. 내부 필드명은 `executable_path`를 사용한다. |
| Arguments | 실행 시 함께 전달할 인수 문자열. 내부 필드명은 `arguments`를 사용한다. |
| Execution Type | 명령 버튼의 실행 방식. 내부 필드명은 `execution_type`을 사용한다. |
| Placeholder Token | 버튼 실행 직전에 워크스페이스 정보, 파일 선택, 폴더 선택, 사용자 입력값으로 대치되는 문자열 |
| Font Setting | 사용자가 선택한 UI 글꼴 이름과 글꼴 크기 |
| UI Language Setting | 애플리케이션 메뉴, 대화상자, 주요 상태/오류 메시지의 표시 언어 |
| Window Layout Setting | 메인 창 크기와 좌우 `워크스페이스`/`명령 그룹` 영역 분할 폭 |
| Settings File | 기본적으로 실행 파일과 같은 폴더에 저장되며, CLI 인자로 다른 경로를 지정할 수 있는 사용자 설정 TOML 파일 |
| App Icon | 프로그램 창에 적용되는 아이콘 자산. Windows는 `icon.ico`, Linux GTK4는 `icon.svg`를 우선 사용하고 없으면 `icon.png`를 사용한다. |
| 정보 창 | 애플리케이션 이름, 버전 정보, GitHub 링크를 표시하는 정보 창 |

## 현재 확정된 규칙

- 애플리케이션 초기 화면에는 좌측 Tree 영역과 우측 명령 그룹 영역이 나란히 표시된다.
- 워크스페이스가 등록되지 않은 초기 상태에서는 트리와 명령 버튼 콘텐츠가 비어 있다.
- 초기 상태 메시지는 `준비됨`이다.
- 좌측 트리에 등록되는 워크스페이스는 `path`, `name`, `Language` 정보로 구성한다.
- 좌측 트리에 등록되는 분류는 Tree의 1차 깊이에만 추가하며 `name`만 가진다.
- 분류 안에 다른 분류를 추가하는 중첩 분류는 지원하지 않는다.
- Tree의 1차 깊이에서는 분류와 최상위 워크스페이스를 섞어 배치할 수 있으며, 분류 우선 또는 워크스페이스 우선 같은 고정 순서를 두지 않는다.
- Tree의 1차 깊이 표시 순서는 `tree_order`에 저장한다. 분류는 `name`, 최상위 워크스페이스는 고유한 `path`로 참조한다.
- `tree_order`가 설정 파일에 없으면 기존 호환 동작으로 분류를 먼저, 그 다음 최상위 워크스페이스를 `workspaces` 순서대로 복원한다.
- 프로그램 시작 시 저장된 선택 항목은 복원하지 않으므로, 등록된 워크스페이스나 분류가 있어도 좌측 Tree의 초기 선택은 없다.
- 분류 안에 있는 워크스페이스는 `tree_order`에 넣지 않고, 해당 분류 하위에서 `workspaces` 배열 순서를 따른다.
- `path`는 로컬 파일 시스템의 폴더 경로이며 워크스페이스 식별의 기준이다.
- `name`은 트리에 표시되는 사용자 친화적 이름이다.
- `Language`는 워크스페이스의 주요 언어를 나타내며 명령 그룹과 명령 버튼 추천 또는 분류에 활용될 수 있다.
- `Language` 값은 사용자 설정의 `languages` 목록에 포함된 값이어야 한다.
- 기본 `languages` 목록은 `Rust`, `TypeScript`, `JavaScript`, `Python`, `Go`, `Java`, `C#`, `C`, `C++`, `Other`다.
- `파일 > UI 언어`는 애플리케이션 표시 언어를 한국어 또는 영어로 즉시 전환한다.
- `파일 > 워크스페이스 언어`는 `languages` 목록을 사용자가 직접 편집하는 화면을 연다.
- 언어 편집값은 줄바꿈 또는 쉼표로 구분할 수 있으며, 중복 값은 대소문자만 다른 경우도 허용하지 않는다.
- 현재 등록된 워크스페이스가 사용하는 `Language`를 새 `languages` 목록에서 제거하려 하면 저장하지 않고 사용자에게 안내한다.
- 폴더 드래그앤드롭으로 워크스페이스를 등록할 때 `path`는 드롭된 폴더 경로로 채우고, `name`은 폴더명으로 기본 설정한다.
- 워크스페이스 추가/편집 대화상자는 저장 전에 `path`가 접근 가능한 폴더인지 확인하고, 접근할 수 없으면 저장하지 않는다.
- 워크스페이스 추가 대화상자에서 폴더를 선택하면 `name`이 비어 있거나 직전 폴더명 기본값과 같을 때 새 폴더명으로 갱신한다.
- 워크스페이스 추가 대화상자에서 폴더를 선택하면 `Language`를 자동 추정해 선택한다. 편집 대화상자의 폴더 선택은 기존 `name`과 `Language`를 자동 변경하지 않는다.
- `Language`는 자동 추정하거나 사용자가 직접 선택할 수 있어야 한다. 자동 추정이 불가능하면 사용자가 선택할 수 있는 상태로 둔다.
- 자동 추정이 불가능한 새 워크스페이스의 초기 선택값은 특정 언어로 오인되지 않도록 `Other`로 둔다.
- 초기 자동 추정은 드롭된 폴더의 최상위 항목만 확인한다.
  - `Cargo.toml` 또는 `*.rs`는 `Rust`로 추정한다.
  - `tsconfig.json`, `*.ts`, `*.tsx`는 `TypeScript`로 추정한다.
  - `package.json`, `*.js`, `*.jsx`는 `JavaScript`로 추정한다.
  - `pyproject.toml`, `requirements.txt`, `setup.py`, `*.py`는 `Python`으로 추정한다.
  - `go.mod`, `*.go`는 `Go`로 추정한다.
  - `pom.xml`, `build.gradle`, `build.gradle.kts`, `*.java`는 `Java`로 추정한다.
  - `*.csproj`, `*.cs`는 `C#`으로 추정한다.
  - `*.c`는 `C`로 추정한다.
  - `CMakeLists.txt`, `*.cpp`, `*.cc`, `*.cxx`, `*.hpp`, `*.hxx`, `*.hh`는 `C++`로 추정한다.
- 자동 추정이 실패한 폴더 드롭은 `path`와 `name`을 채운 추가 대화상자를 열고, 사용자가 `Language`를 선택할 수 있게 한다.
- 같은 `path`를 가진 워크스페이스는 중복 등록하지 않는다. 중복 판정은 Windows 기준 동작처럼 경로 양끝 공백과 끝 경로 구분자를 제외하고 ASCII 대소문자를 구분하지 않는다.
- 등록된 워크스페이스 목록은 설정 파일에 저장하고, 프로그램 시작 시 복원한다.
- 좌측 Tree에서 워크스페이스 노드를 마우스로 끌어 다른 워크스페이스 앞 또는 뒤로 놓으면 `workspaces` 저장 순서를 변경한다.
- 좌측 Tree에서 최상위 워크스페이스 노드를 선택하고 `Ctrl+Up` 또는 `Ctrl+Down`을 누르면 `tree_order` 안에서 분류와 워크스페이스를 포함한 앞뒤 항목과 한 칸 이동한다.
- 좌측 Tree에서 분류 안의 워크스페이스 노드를 선택하고 `Ctrl+Up` 또는 `Ctrl+Down`을 누르면 같은 분류 안에서 한 칸 위 또는 아래로 이동한다.
- 좌측 Tree에서 설정된 분류와 매칭되는 분류 안의 워크스페이스 노드를 선택하고 `Ctrl+Left`를 누르면 해당 워크스페이스의 `category`를 해제해 Tree의 1차 깊이로 이동한다.
- 좌측 Tree에서 분류 노드를 선택하고 `Ctrl+Up` 또는 `Ctrl+Down`을 누르면 `tree_order` 안에서 분류와 워크스페이스를 포함한 앞뒤 항목과 한 칸 이동한다.
- `워크스페이스 > 위로`와 `워크스페이스 > 아래로`는 선택된 워크스페이스 또는 분류에 같은 한 칸 이동 규칙을 적용한다.
- `워크스페이스 > 편집`은 선택된 워크스페이스는 워크스페이스 편집창으로, 선택된 분류는 분류 이름 입력창으로 편집한다.
- 좌측 Tree에서 워크스페이스 또는 분류 항목을 더블클릭하면 `워크스페이스 > 편집`과 같은 편집 동작을 실행한다.
- 좌측 Tree에서 워크스페이스 또는 분류 항목을 우클릭하면 해당 항목을 선택하고 컨텍스트 메뉴를 표시한다.
- 트리 컨텍스트 메뉴는 편집, 위로, 아래로, 워크스페이스 추가, 분류 추가, 삭제를 제공하며, 삭제는 워크스페이스 또는 분류가 선택된 경우 활성화한다.
- 분류 이름을 변경하면 해당 분류를 참조하는 하위 워크스페이스의 `category` 값과 `tree_order`의 분류 참조도 함께 갱신한다.
- 분류를 삭제하면 분류만 제거하고, 해당 분류에 속한 워크스페이스는 삭제하지 않고 Tree의 1차 깊이로 이동한다.
- 워크스페이스를 분류 노드 위에 드롭하면 해당 워크스페이스의 `category` 소속을 그 분류 이름으로 저장하고, 트리에서는 해당 분류의 하위 워크스페이스로 표시한다.
- 좌측 Tree에서 워크스페이스를 드래그하는 동안 유효한 드롭 대상 노드가 있으면 대상 노드 배경을 현재 테마와 대비되는 피드백 색으로 표시하고, 대상에서 벗어나거나 드롭/취소하면 원래 색으로 복원한다.
- `category`가 없거나 저장된 `category` 이름과 일치하는 분류가 없으면 워크스페이스는 Tree의 1차 깊이에 표시한다.
- 최상위 워크스페이스를 다른 최상위 워크스페이스 앞 또는 뒤로 드롭하면 `tree_order`에서 해당 위치로 이동한다.
- 설정 파일에서 워크스페이스 언어 필드명은 `Language` 대소문자를 그대로 유지한다.
- 명령 버튼은 `button_name`, `executable_path`, `arguments`, `execution_type` 정보로 구성한다.
- `button_name`은 버튼에 표시되는 사용자 친화적 이름이다.
- `executable_path`는 실행할 프로그램 경로 또는 실행 대상 명령이다.
- `arguments`는 실행 직전에 치환 토큰을 실제 값으로 대체한 뒤 실행 방식에 맞게 전달한다.
- `execution_type`은 `shell_api` 또는 `external_terminal` 중 하나다.
- `shell_api` 실행 방식은 플랫폼 실행 백엔드를 통해 `executable_path`를 직접 실행하고, 치환된 `arguments`를 인자로 전달한다. Windows는 Shell API, Linux GTK4 백엔드는 POSIX shell 경유 실행을 사용한다.
- `external_terminal` 실행 방식은 외부 터미널을 열고 선택된 워크스페이스의 `path`를 작업 디렉터리로 사용해 명령을 실행한다.
- `external_terminal` 실행 방식은 선택된 워크스페이스가 없거나 워크스페이스 `path`에 접근할 수 없으면 실행하지 않는다.
- `shell_api` 실행 방식도 `arguments`에 `{path}`, `{name}`, `{Language}` 같은 워크스페이스 토큰을 사용하면 선택된 워크스페이스가 필요하며, 해당 워크스페이스 `path`에 접근할 수 없으면 실행하지 않는다.
- 명령 버튼의 `arguments`에는 `{path}`, `{name}`, `{selectfile}`, `{selectdir}`, `{inputtext}`, `{Language}` 토큰을 사용할 수 있다.
- 토큰은 버튼 실행 시점에 대치한다.
- `{path}`는 선택된 워크스페이스의 `path`로 대치한다.
- `{name}`은 선택된 워크스페이스의 `name`으로 대치한다. `name`의 기본값은 `path`의 마지막 폴더명이며, 사용자가 편집할 수 있다.
- `{selectfile}`은 파일 선택창에서 사용자가 선택한 파일 경로로 대치한다.
- `{selectdir}`은 폴더 선택창에서 사용자가 선택한 폴더 경로로 대치한다.
- `{inputtext}`는 텍스트 입력창에서 사용자가 입력한 문자열로 대치한다.
- `{Language}`는 선택된 워크스페이스의 `Language`로 대치한다.
- 치환 과정에서 사용자가 파일 선택, 폴더 선택, 텍스트 입력을 취소하면 버튼 실행은 취소한다.
- 앱 소유 모달 대화상자를 닫으면 메인 창을 다시 활성화하고 전면 상태를 유지한다.
- 모달 대화상자의 저장/취소/닫기 키 입력은 대화상자 안에서 처리되어야 하며, 대화상자가 닫힌 뒤 부모 창의 트리나 버튼 활성화 이벤트로 다시 전달되지 않아야 한다.
- 치환된 토큰 값은 플랫폼별 명령행 규칙에 따라 하나의 인자로 보존되도록 이스케이프한다. Windows는 Windows 명령행 규칙을, Linux는 POSIX shell quoting 규칙을 따른다.
- 등록된 명령 그룹과 명령 버튼 목록은 설정 파일에 저장하고, 프로그램 시작 시 복원한다.
- 명령 그룹은 설정 파일의 `command_tabs` 배열에 저장하며, 각 그룹은 `name`과 소속 `buttons`를 가진다.
- 명령 그룹 선택은 우측 영역 상단의 드롭다운 목록으로 제공한다.
- 명령 그룹이 없으면 우측 영역의 드롭다운은 빈 상태로 비활성화된다.
- `명령 그룹 > 삭제`는 선택된 명령 그룹을 삭제하며, 해당 그룹에 포함된 명령 버튼도 다른 그룹으로 이동하지 않고 함께 설정에서 제거한다.
- 명령 그룹 삭제 확인 문구에는 함께 삭제될 명령 버튼 개수를 표시한다.
- `명령 그룹 > 위로`는 선택된 명령 그룹을 드롭다운 순서에서 한 칸 위로 이동한다.
- `명령 그룹 > 아래로`는 선택된 명령 그룹을 드롭다운 순서에서 한 칸 아래로 이동한다.
- `명령 그룹 > 위로`와 `명령 그룹 > 아래로`는 선택된 명령 그룹이 해당 방향으로 이동 가능한 경우에만 활성화한다.
- `명령 > 추가`는 선택된 명령 그룹이 있을 때만 활성화하며, 새 명령 버튼은 선택된 그룹의 `buttons`에 저장한다.
- `명령 > 실행`, `명령 > 편집`, `명령 > 앞으로`, `명령 > 뒤로`, `명령 > 삭제`는 선택된 명령 버튼이 있을 때만 활성화한다.
- `명령 > 앞으로`는 선택된 명령 버튼을 같은 명령 그룹 안에서 한 칸 앞 위치로 이동하며, 첫 번째 명령 버튼에서는 비활성화한다.
- `명령 > 뒤로`는 선택된 명령 버튼을 같은 명령 그룹 안에서 한 칸 뒤 위치로 이동하며, 마지막 명령 버튼에서는 비활성화한다.
- 명령 버튼은 선택된 명령 그룹의 드롭다운 아래 콘텐츠 영역에 플랫폼 네이티브 컨트롤 목록으로 표시한다. Windows는 Win32 `BUTTON` 컨트롤 격자, Linux는 GTK4 `FlowBox` wrapping grid를 사용한다. 버튼 폭, 높이, 내부 시작 여백, 행/열 간격은 현재 UI 글꼴 크기에 맞춰 같은 기본 규칙으로 조정한다.
- 명령 버튼이 현재 버튼 표시 영역 높이를 넘으면 앱 소유 페이지 창에 세로 스크롤을 제공해 마지막 버튼까지 접근할 수 있게 한다.
- 명령 버튼 세로 스크롤은 버튼 행 단위로 정렬해 중간 픽셀 위치에서 버튼 컨트롤이 잘린 상태로 멈추지 않게 한다.
- 마지막 명령 버튼 행이 페이지 하단에 걸리는 경우에는 하단 빈 여백을 허용해 마지막 명령 버튼 전체가 보이도록 한다.
- 명령 버튼 표시 텍스트가 버튼 폭보다 길면 왼쪽 시작 부분을 유지하고, 버튼 경계를 넘는 오른쪽 텍스트는 숨긴다.
- 긴 명령 버튼 텍스트는 같은 행의 오른쪽 버튼 영역이나 텍스트를 침범하지 않아야 한다.
- 명령 버튼 위에 마우스를 올리면 해당 명령 버튼의 전체 `button_name`을 툴팁으로 표시한다.
- 명령 버튼 클릭은 현재 선택된 워크스페이스와 명령 버튼을 확인한 뒤 실행 동작으로 처리한다.
- 좌측 Tree에서 분류 노드가 선택된 상태에서 명령 버튼을 클릭하면 실행하지 않고 경고나 오류창 없이 무시한다.
- 명령 버튼을 우클릭하거나 명령 버튼에 포커스가 있을 때 `Menu` 키 또는 `Shift+F10`을 누르면 해당 버튼을 선택하고 컨텍스트 메뉴를 표시한다.
- 명령 버튼 컨텍스트 메뉴는 실행, 편집, 앞으로, 뒤로, 명령 추가, 삭제를 제공하며, 실행은 좌측 Tree에서 분류 노드가 선택된 상태에서는 비활성화한다.
- 명령 버튼을 마우스로 끌어 같은 명령 그룹 안의 다른 명령 버튼 위에 놓으면 드롭 대상 버튼이 가리키는 순서 위치로 이동한다.
- 명령 버튼 드래그 중 유효한 드롭 대상 버튼 위에 있으면 대상 버튼 배경을 현재 테마와 대비되는 피드백 색으로 표시하고, 대상에서 벗어나거나 드롭/취소하면 원래 색으로 복원한다.
- 명령 버튼 드래그 이동이 완료되면 변경된 `buttons` 순서를 설정 파일에 저장하고, 이동한 버튼을 선택 상태로 유지한다.
- 실행 직전 `arguments`의 토큰을 치환하고, 사용자 선택 또는 입력이 취소되면 실행 전체를 취소한다.
- `명령 > 추가`와 `명령 > 편집`은 같은 버튼 편집창을 사용한다.
- 버튼 편집창의 `적용`은 창을 닫지 않고 현재 값을 선택된 명령 그룹의 명령 버튼 목록과 설정 파일에 즉시 저장한다.
- 버튼 편집창의 `저장`은 현재 값을 저장한 뒤 창을 닫고, `취소`는 마지막으로 적용되지 않은 편집 내용을 버리고 창을 닫는다.
- 버튼 편집창에서 `arguments`에 알 수 없는 `{...}` 토큰이 있으면 저장하지 않고 오류 메시지를 표시한다.
- 토큰 삽입 버튼은 `arguments` 입력의 현재 커서 위치에 토큰을 삽입하고 `arguments` 입력 포커스를 유지한다.
- 사용자는 시스템에 설치된 글꼴을 선택해 애플리케이션 UI 글꼴을 변경할 수 있다.
- 글꼴 목록은 설치 글꼴이 많은 환경에서도 세로 스크롤로 끝까지 탐색할 수 있어야 한다.
- 사용자는 UI 글꼴 크기를 변경할 수 있다.
- 글꼴 이름, 글꼴 크기, 테마, UI 언어는 사용자 설정으로 저장한다.
- 메인 창 크기와 좌측 Tree 영역 폭은 사용자 설정으로 저장하고, 다음 실행 시 복원한다.
- 우측 명령 그룹 영역 폭은 복원된 창 크기와 좌측 Tree 영역 폭에서 계산한다.
- 테마는 `system`, `light`, `classic-dark`, `sepia-teal`, `graphite`, `forest`, `steel-blue` 중 하나다.
- 기본 테마는 `graphite`다.
- UI 언어는 `en` 또는 `ko` 중 하나다.
- 기본 UI 언어는 영어 UI 적용 상태인 `en`이다.
- UI 언어를 전환하면 메뉴, 테마/UI 언어 하위 메뉴, 주요 대화상자 라벨, 주요 상태/오류 메시지를 새 언어로 표시한다.
- UI 언어별 도메인 용어와 도메인/인수 오류의 사용자 메시지는 `domain` 경계에서 제공하고, 플랫폼 UI 구현은 현재 언어를 넘겨 표시만 담당한다.
- `system`은 OS 기본 색과 컨트롤 렌더링을 우선 사용하고, 나머지 테마는 j3TreeText의 색상 계열을 참고해 창 배경, TreeView 배경/텍스트/라인, 우측 명령 버튼 콘텐츠 배경, 일반 컨트롤 배경을 적용한다.
- 폴더 드래그앤드롭 피드백 색은 현재 테마의 TreeView 텍스트와 라인 대비를 유지해야 한다.
- UI 글꼴 기본값은 시스템 기본 UI 글꼴 정책에 맞춰 `Segoe UI`와 `12` pt로 둔다.
- UI 글꼴 크기는 레이아웃이 깨지지 않도록 `9` pt 이상 `20` pt 이하로 제한한다.
- 사용자가 선택할 수 있는 글꼴 크기 목록은 `9`, `10`, `11`, `12`, `13`, `14`, `16`, `18`, `20`이다.
- 설정 파일의 `view.font_size`도 위 선택 목록 값만 복원한다. 목록 밖 값은 기본값 `12`로 복원하고 경고를 남긴다.
- 저장된 글꼴이 다음 실행 시 시스템 설치 글꼴 목록에 없으면 기본 글꼴을 사용하고 복원 경고를 남긴다.
- Linux에서 Windows 기본 글꼴인 `Segoe UI`가 설치되어 있지 않은 기본 설정 상태는 경고로 보지 않고 GTK/Pango의 글꼴 fallback에 맡긴다.
- Linux 글꼴 설정 대화상자에서도 현재 값이 Windows 기본 글꼴 `Segoe UI`이면 시스템 설치 목록에 없더라도 현재 선택값으로 표시해, 아무 변경 없이 적용할 때 설정이 다른 설치 글꼴로 바뀌지 않게 한다.
- 글꼴 변경은 좌측 Tree, 우측 명령 그룹 드롭다운, 명령 버튼, 일반 대화상자 텍스트에 적용하며, 적용 후 레이아웃을 다시 계산한다.
- Windows 커스텀 대화상자는 현재 UI 글꼴 크기, 실제 적용된 Win32 `HFONT` 높이, 창 DPI를 같은 기준으로 반영해 창 크기와 내부 컨트롤 좌표/크기를 계산한다. 설정 파일 위치나 debug/release 빌드 차이로 저장된 글꼴 크기와 실제 폰트 핸들이 어긋나도, 지원 범위인 9~20pt에서 라벨, 입력칸, 버튼 텍스트가 잘리거나 겹치지 않아야 한다.
- 설정 파일은 기본적으로 프로그램 실행 파일과 같은 폴더에 `실행 파일이름.toml` 형식으로 저장한다.
  - 예: `j3DevHelper.exe`의 기본 설정 파일은 `j3DevHelper.toml`이다.
- 프로그램 시작 시 실행 파일 뒤에 설정 파일 경로 인자 하나를 넘기면 해당 TOML 파일을 읽고, 이후 설정 저장도 같은 경로에 기록한다.
  - 예: `j3devhelper ./j3devhelper-linux.toml`
- 설정 파일 경로 인자를 2개 이상 넘기면 시작 오류로 처리한다.
- 저장소에는 Linux에서 바로 참고할 수 있는 예시 설정 파일 `j3devhelper-linux.toml`을 둔다. 이 예시는 현재 `j3DevHelper` 워크스페이스와 VS Code, Android Studio, 파일 관리자, Rust Cargo 명령 버튼을 포함한다.
- 프로그램 시작 시 설정 파일이 있으면 읽어서 UI 설정을 복원한다.
- 설정 파일이 없거나 읽을 수 없거나 TOML 파싱에 실패하면 기본 설정으로 시작하고, 사용자에게 표시 가능한 경고 메시지를 상태로 남긴다.
- 설정 파일에 존재하지 않는 선택 항목은 기본값을 사용한다.
- 설정 파일에서 필수 필드가 빠졌거나 비어 있는 워크스페이스는 건너뛰고 복원 실패 상태를 기록한다.
- 설정 파일에서 같은 `path`를 가진 워크스페이스가 여러 개 있으면 첫 항목만 복원하고 이후 항목은 건너뛰며 복원 실패 상태를 기록한다. 이때 `path` 중복 판정도 Windows 기준처럼 ASCII 대소문자를 구분하지 않는다.
- 설정 파일에서 `languages`가 없으면 기본 `languages` 목록을 사용한다.
- 설정 파일에서 `languages`가 문자열 배열이 아니거나 비어 있거나 중복이 있으면 기본 `languages` 목록을 사용하고 복원 실패 상태를 기록한다.
- 설정 파일에서 워크스페이스 `Language` 값이 복원된 `languages` 목록에 없으면 해당 워크스페이스는 건너뛰고 복원 실패 상태를 기록한다.
- 설정 파일에서 필수 필드가 빠졌거나 비어 있는 명령 버튼은 건너뛰고 복원 실패 상태를 기록한다.
- 설정 파일에서 명령 버튼의 `execution_type` 값이 `shell_api` 또는 `external_terminal`이 아니면 해당 명령 버튼은 건너뛰고 복원 실패 상태를 기록한다.
- 설정 저장 실패는 조용히 무시하지 않고 호출자에게 오류로 전달한다.
- Windows에서는 `icon.ico`를 실행 파일 아이콘과 프로그램 창 아이콘에 모두 적용한다. Linux GTK4에서는 실행 파일 폴더 또는 현재 작업 폴더의 `icon.svg`를 우선 GTK 아이콘 테마 검색 경로에 추가하고, 없으면 `icon.png`를 창 아이콘으로 적용한다. GTK default icon name도 같은 이름으로 설정하고, GTK titlebar 왼쪽에도 같은 아이콘을 표시해 제목 표시줄과 창 전환 UI가 같은 앱 아이콘을 참조하도록 한다.
- Linux GTK4 application id와 desktop entry 표시 이름/코멘트는 `io.github.edgarp9.j3DevHelper`다. application id에 대문자가 있으므로 설치 시 Plasma fallback용 `io.github.edgarp9.j3devhelper` alias desktop entry와 alias icon도 함께 설치한다.

## 설정 저장 형식

사용자 설정 파일은 TOML 형식이며 다음 최상위 항목을 가진다.

| 항목 | 형식 | 설명 |
| --- | --- | --- |
| `view` | table | UI 표시 설정 |
| `languages` | array of strings | 워크스페이스 `Language` 선택 목록 |
| `tree_order` | array of tables | Tree 1차 깊이에 표시되는 분류와 최상위 워크스페이스의 혼합 순서 |
| `categories` | array of tables | Tree 1차 깊이에 표시될 수 있는 분류 목록 |
| `workspaces` | array of tables | 등록된 워크스페이스 목록 |
| `command_tabs` | array of tables | 명령 그룹 목록. 기존 설정 호환을 위해 키 이름은 유지한다. |
| `command_tabs.buttons` | array of tables | 각 명령 그룹에 속한 명령 버튼 목록 |

`view`는 다음 필드를 가진다.

| 필드 | 필수 여부 | 기본값 |
| --- | --- | --- |
| `font_family` | 선택 | `Segoe UI` |
| `font_size` | 선택 | `12` |
| `theme` | 선택 | `graphite` (`system`, `light`, `classic-dark`, `sepia-teal`, `graphite`, `forest`, `steel-blue`) |
| `ui_language` | 선택 | `en` (`en`, `ko`) |
| `window_width` | 선택 | 마지막 정상 종료 시 OS가 보고한 메인 창 폭(px). 없으면 기본 창 폭 |
| `window_height` | 선택 | 마지막 정상 종료 시 OS가 보고한 메인 창 높이(px). 없으면 기본 창 높이 |
| `tree_panel_width` | 선택 | 마지막 정상 종료 시 좌측 Tree 영역 폭(px). 없으면 기본 폭 |

`categories`의 각 항목은 다음 필드를 가진다.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| `name` | 필수 | 분류 표시 이름 |

`workspaces`의 각 항목은 다음 필드를 가진다.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| `path` | 필수 | 워크스페이스 폴더 경로 |
| `name` | 필수 | 트리 표시 이름 |
| `Language` | 필수 | 워크스페이스 언어. TOML 키 대소문자를 반드시 `Language`로 유지한다. |
| `category` | 선택 | 소속 분류 이름. 없으면 최상위 워크스페이스로 표시한다. |

`tree_order`의 각 항목은 다음 필드를 가진다.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| `type` | 필수 | `category` 또는 `workspace` |
| `name` | `type = "category"`일 때 필수 | 참조할 분류 이름 |
| `path` | `type = "workspace"`일 때 필수 | 참조할 최상위 워크스페이스 경로 |

`tree_order`는 분류와 `category`가 없는 워크스페이스만 참조한다. 분류 하위 워크스페이스는 해당 분류 자식으로 표시하며 `tree_order`에 저장하지 않는다.

`command_tabs`의 각 항목은 다음 필드를 가진다.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| `name` | 필수 | 명령 그룹 표시 이름 |
| `buttons` | 선택 | 명령 그룹에 속한 명령 버튼 목록 |

`command_tabs.buttons`의 각 항목은 다음 필드를 가진다.

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| `button_name` | 필수 | 명령 버튼 표시 이름 |
| `executable_path` | 필수 | 실행할 프로그램 경로 또는 명령 |
| `arguments` | 선택 | 실행 시 전달할 인수. 생략하면 빈 문자열로 복원한다. |
| `execution_type` | 필수 | `shell_api` 또는 `external_terminal`만 허용한다. |

## 현재 구현 결정

- Cargo 기반 Rust 애플리케이션으로 초기 프로젝트를 구성한다.
- Windows Win32 호출은 `windows-sys`를 사용하고, Windows 전용 구현은 `infra::win32`에 둔다.
- Linux UI는 GTK4를 사용하고, Linux 전용 구현은 `infra::gtk4`에 둔다.
- 진입점은 `src/main.rs`, 실행 흐름과 초기 상태는 `src/app.rs`, 초기 UI 규칙과 메뉴 정의는 `src/domain.rs`에 둔다.
- 워크스페이스 트리 이동/드롭, 명령 그룹 이동, 명령 버튼 이동/드롭의 순수 목적지 계산은 `domain::navigation` 공통 규칙으로 두고 Windows/Linux UI 백엔드가 같은 함수를 사용한다.
- 메인 윈도우 제목은 `j3DevHelper`, 초기 크기는 `484 x 613` px로 둔다. Windows는 순수 Win32 일반 최상위 창, Linux는 GTK4 `ApplicationWindow`로 생성한다.
- `system` 테마에서는 시스템 기본 타이틀바와 시스템 `COLOR_WINDOW` 기반 배경 브러시를 사용한다.
- 메뉴 바는 현재 UI 언어에 따라 `File`/`파일`, `Workspace`/`워크스페이스`, `Command Group`/`명령 그룹`, `Command`/`명령` 순서로 생성한다.
- `File`/`파일` 메뉴는 `Font`/`글꼴`, `Theme`/`테마`, `UI Language`/`UI 언어`, `Workspace Languages`/`워크스페이스 언어`, `About`/`정보`, `Exit`/`종료` 항목을 제공한다.
- `파일 > 종료`는 선택 시 메인 창 닫기 요청과 같은 종료 경로를 따른다.
- `파일 > 정보`는 앱 이름과 현재 패키지 버전을 한 줄(`j3DevHelper  v{version}`)로 표시하고, `https://github.com/edgarp9` 링크를 표시하는 정보 창을 메인 윈도우 중앙에 연다.
- `워크스페이스` 메뉴는 `추가`, `분류 추가`, `편집`, `위로`, `아래로`, `삭제` 항목을 가진다.
- `명령 그룹` 메뉴는 `추가`, `이름 변경`, `위로`, `아래로`, `삭제` 항목을 가진다.
- 이번 단계에서는 최상위 워크스페이스의 추가, 편집, 삭제와 1차 깊이 분류 추가, 이름 편집, 삭제를 구현한다.
- `워크스페이스 > 분류 추가`는 분류 `name`을 입력받아 Tree의 1차 깊이에 폴더형 노드로 추가한다.
- 분류 추가/편집과 명령 그룹 추가/이름 변경의 단일 텍스트 입력창은 `Enter`를 저장 동작으로 처리한다.
- 분류는 `path`와 `Language`를 갖지 않으며, 분류 안에 다른 분류를 추가하지 않는다.
- `워크스페이스 > 추가`는 폴더 선택, `name` 입력, `Language` 선택을 제공한다.
- 메뉴로 워크스페이스를 추가할 때 사용자가 폴더를 선택하면 `name` 기본값은 선택한 폴더명으로 설정한다.
- `워크스페이스 > 편집`은 선택된 워크스페이스의 `path`, `name`, `Language`를 표시하고 수정할 수 있게 하며, 선택된 분류는 `name`만 수정할 수 있게 한다.
- `워크스페이스 > 삭제`는 즉시 삭제하지 않고 확인 절차를 거친 뒤 삭제한다.
- `워크스페이스 > 위로`와 `워크스페이스 > 아래로`는 현재 선택된 워크스페이스 또는 분류가 해당 방향으로 이동 가능한 경우에만 활성화한다. 최상위 워크스페이스와 분류는 같은 `tree_order` 안에서 이동한다.
- 트리/명령 그룹/명령 버튼 메뉴 활성화와 실제 이동 실행은 같은 공통 이동 목적지 규칙을 사용해야 한다.
- 선택된 워크스페이스 또는 분류가 없으면 `워크스페이스 > 삭제`는 비활성화한다. `워크스페이스 > 편집`은 워크스페이스 또는 분류 노드가 선택된 경우 활성화한다.
- 트리 노드 기본 표시 텍스트는 `name`만 사용하고, `path`와 `Language`는 툴팁과 편집 화면에서 확인한다.
- 워크스페이스 변경 사항은 사용자 설정 파일에 즉시 저장하고 다음 실행 시 복원한다.
- 초기 상태 메시지는 `AppState` 내부에 `준비됨`으로 보관한다.
- 초기 콘텐츠는 플랫폼 네이티브 컨트롤로 구성한다. Windows는 좌측 `SysTreeView32`, 우측 명령 그룹 선택용 드롭다운과 명령 버튼 표시용 앱 소유 페이지 창을 둔다. Linux는 워크스페이스 트리용 GTK4 `ListBox`, 명령 버튼용 `FlowBox`, `ComboBoxText`, `Paned` 기반 레이아웃을 둔다. 좌우 영역 상단에 별도 라벨은 표시하지 않는다.
- 메인 콘텐츠 바깥 여백은 약 8 px, 메뉴 아래 간격은 약 6 px, 좌우 패널 간격은 약 10 px로 두어 클라이언트 영역의 빈 공간을 줄인다.
- 좌측 트리 컨트롤 폭은 약 160 px로 유지하고, 우측 명령 그룹 드롭다운과 버튼 페이지는 남은 클라이언트 폭을 채운다. `WM_SIZE` 처리에서 두 영역의 높이를 다시 계산한다.
- `WM_GETMINMAXINFO`에서 현재 글꼴 크기와 DPI를 반영한 최소 창 크기를 지정한다. 최소 클라이언트 크기는 좌측 트리 최소 폭 72 px, 패널 간격, 우측 명령 그룹 영역 최소 폭 116 px, 메인 패널 최소 높이 160 px를 DPI 스케일한 값으로 계산한다.
- Windows는 저장소의 `icon.ico`를 앱 아이콘 배포 자산으로 사용한다. Linux GTK4는 저장소의 `icon.svg`를 창 아이콘과 desktop entry 아이콘 배포 자산으로 우선 사용하고, 없으면 `icon.png`를 사용한다.
- `build.rs`는 `winresource`를 기본 기능 없이 사용해 Windows 타깃 빌드에서 `icon.ico`를 실행 파일 리소스 ID `1`로 임베드한다.
- `build.rs`는 실행 파일 매니페스트도 함께 임베드해 Common Controls v6와 Per-Monitor DPI Awareness V2를 선언한다.
- Windows 실행 시 창 아이콘은 임베드된 리소스 ID `1`을 우선 로드하고, 개발 실행 fallback으로 실행 파일 폴더 또는 현재 작업 폴더의 `icon.ico`를 로드한다.
- Linux 실행 시 창 아이콘은 실행 파일 폴더 또는 현재 작업 폴더의 `icon.svg`를 우선 GTK 아이콘 테마 검색 경로에 추가해 적용하고, 없으면 `icon.png`를 사용한다. 로컬 아이콘을 찾지 못하면 설치된 hicolor 아이콘의 application id를 창/default icon name으로 사용한다.
- Linux `--install`은 XDG user data 경로 아래에 desktop entry와 hicolor 앱 아이콘을 설치한다. 설치는 같은 내용을 다시 쓰지 않는 idempotent 동작이며, `io.github.edgarp9.j3DevHelper`에 대문자가 있으므로 lowercase alias desktop entry/icon도 함께 설치한다. alias desktop entry에는 앱 메뉴 중복을 막기 위해 `NoDisplay=true`를 둔다.
- Linux `--uninstall`은 현재 application id와 lowercase alias의 desktop entry/icon만 제거한다.
- Windows 실행 파일은 debug와 release 빌드 모두 GUI subsystem으로 빌드해 앱 시작 시 별도 콘솔 창을 표시하지 않는다.
- 콘솔이 없는 Windows 실행에서 시작 오류가 발생하면 표준 오류 출력 대신 오류 대화상자로 사용자에게 표시한다.
- 실행 시 프로세스 DPI awareness를 Per-Monitor V2로 설정하고, 저장된 메인 창 폭/높이는 실제 창 픽셀로 복원한다. 현재 DPI는 메인 레이아웃, 최소 창 크기, 대화상자 크기와 컨트롤 좌표, UI 글꼴 픽셀 높이 계산에 반영한다.
- `WM_DPICHANGED`를 받으면 새 DPI로 글꼴과 레이아웃을 다시 만들고 Windows가 제공한 권장 창 위치/크기를 적용한다.
- Windows `external_terminal`은 `%ComSpec%` 또는 `cmd.exe`를 `/D /S /K`로 열어 명령 종료 후 창을 유지한다.
- Windows `external_terminal`의 `executable_path`는 전체 경로뿐 아니라 `cmd.exe`에서 PATH로 해석 가능한 명령 이름도 허용한다.
- Windows `external_terminal`은 `cmd.exe /D /S /K` 형식으로 명령을 전달해, 공백이 포함된 실행 파일 경로와 인자를 Windows 명령행 규칙에 맞게 보존한다.
- Linux `external_terminal`은 `$TERMINAL` 또는 일반 터미널 에뮬레이터를 찾아 선택 워크스페이스에서 `sh -lc`로 명령을 실행하고 종료 상태 확인 후 닫히도록 한다.
- `파일 > UI 언어`는 한국어/영어 하위 메뉴를 제공하고, 선택 시 `view.ui_language`에 즉시 저장한 뒤 메뉴와 이후 대화상자 표시 언어를 갱신한다.
- `파일 > 워크스페이스 언어`는 현재 `languages` 목록을 편집창에 표시하고, 저장 시 설정 파일에 즉시 반영한다.

## 현재 통합 점검

- 초기 화면은 메뉴 바 아래 좌측 빈 워크스페이스 트리, 우측 빈 명령 그룹 드롭다운으로 시작한다.
- 우측 명령 버튼은 드롭다운 아래 콘텐츠 영역에 배치하며 버튼 목록이 높이를 넘으면 세로 스크롤한다.
- 사용자는 좌측 `워크스페이스` 영역과 우측 `명령 그룹` 영역 사이의 간격을 마우스로 드래그해 좌측 트리 폭을 조절할 수 있다.
- 좌우 영역 조절은 현재 창에서 즉시 반영하며, 좌측 트리 최소 폭과 우측 명령 그룹 영역 최소 폭을 침범하지 않는다. 창을 최소 폭으로 줄일 때는 현재 좌측 Tree 폭도 최소 폭까지 자동으로 축소될 수 있다.
- 마지막 정상 종료 시점의 메인 창 크기와 좌측 Tree 영역 폭은 OS가 보고한 실제 픽셀 값으로 설정 파일에 저장하며, 다음 실행 시 같은 값으로 복원한다. DPI 변경 중에는 `WM_DPICHANGED` 권장 창 위치/크기와 레이아웃 재계산을 따른다.
- 메뉴는 UI 언어에 맞춰 `File`/`파일`, `Workspace`/`워크스페이스`, `Command Group`/`명령 그룹`, `Command`/`명령` 순서이며, `File`/`파일`에는 글꼴, 테마, UI 언어, 워크스페이스 언어, 정보, 종료 항목을 둔다.
- `파일 > 정보`는 `Cargo.toml`의 패키지 버전에 맞춘 앱 버전을 앱 이름과 같은 줄에 두고, `https://github.com/edgarp9` 링크를 메인 윈도우 중앙 대화상자로 표시한다.
- 최상위 워크스페이스 추가, 편집, 삭제, 마우스 드래그 순서 변경, 1차 깊이 분류 추가와 이름 편집, 분류와 최상위 워크스페이스의 혼합 `tree_order`, 워크스페이스의 분류 드롭 배치, 폴더 드래그앤드롭 등록, 중복 `path` 방지, 툴팁의 `path`와 `Language` 표시는 구현되어 있다.
- 명령 그룹 추가, 이름 변경, 위로, 아래로, 삭제와 명령 실행, 추가, 편집, 앞으로, 뒤로, 삭제, 마우스 드래그 순서 변경은 설정 파일과 연동되어 있다.
- 버튼 편집창은 알 수 없는 `{...}` 인수 토큰을 저장 전에 거부하고, 토큰 삽입 버튼은 현재 커서 위치에 토큰을 삽입한다.
- 설정 파일은 실행 파일과 같은 폴더의 `실행 파일 이름.toml`로 저장하며, 읽기/파싱/저장 실패는 상태 메시지 또는 대화상자로 표시한다.
- 글꼴 설정은 설치 글꼴 목록과 허용 크기 목록을 사용하고, 적용 시 메인 컨트롤과 대화상자 컨트롤을 다시 계산한다. Windows는 DPI 기준 Win32 글꼴과 레이아웃을, Linux는 Pango/GTK CSS 기반 글꼴 적용을 사용한다. `파일 > 테마` 하위 메뉴는 선택 테마를 체크 표시한다.
- UI 언어 하위 메뉴는 한국어/영어를 제공하며 현재 선택 언어를 체크 표시한다.
- 워크스페이스 언어 화면은 줄바꿈 또는 쉼표로 구분된 언어 목록을 저장하고, 워크스페이스 추가/편집 대화상자의 `Language` 콤보박스는 이 목록을 사용한다.
- 테마 설정은 `system`, `light`, `classic-dark`, `sepia-teal`, `graphite`, `forest`, `steel-blue`를 지원하고, 선택 즉시 창 배경, TreeView 색상, 우측 명령 버튼 콘텐츠 배경, 컨트롤 다크 모드 선호값에 적용한다.

## Linux GTK4 지원

- Windows UI 백엔드는 기존 `infra::win32` Win32 구현을 유지한다.
- Linux UI 백엔드는 `infra::gtk4` GTK4 구현을 사용한다.
- `src/app.rs`는 같은 `MainWindowSpec`과 `AppState`를 만든 뒤 target OS에 맞는 UI 백엔드로 넘긴다.
- `src/infra/mod.rs`는 Windows와 Linux UI 모듈을 `cfg`로 분리해 한 코드베이스에서 두 플랫폼을 빌드할 수 있게 한다.
- Linux GTK4 구현은 워크스페이스/분류 관리, 명령 그룹/명령 버튼 관리, 토큰 치환, 파일/폴더/텍스트 입력, 테마, UI 언어, 글꼴, 설정 저장/복원을 같은 도메인 규칙으로 처리한다.
- 플랫폼별 기능 비교와 남은 차이는 [Windows/Linux Platform Equivalence](platform-equivalence.md)에 기록한다.

## 미정 사항

- 자동 추정 규칙을 사용자가 추가한 임의의 새 언어까지 확장하는 방식
- 외부 터미널을 `cmd.exe` 외 다른 터미널로 선택하는 설정
